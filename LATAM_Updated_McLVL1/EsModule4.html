Módulo 4 de 5
🌲 Bosque Mecánico

Automatización viva: los estudiantes aplican cada habilidad aprendida en los Módulos 1 a 3 para construir un sistema agrícola integral automatizado de extremo a extremo: el Agente siembra las semillas, los circuitos de Redstone cosechan de manera autónoma al madurar los cultivos, y las Tolvas clasifican y almacenan el producto final.

Objetivos del Módulo 4: Programar al Agente como un jardinero automatizado mediante bucles y comandos de siembra · Construir circuitos condicionales de Redstone para la cosecha automática (Observador → Pistón) · Implementar sistemas de almacenamiento y clasificación de elementos mediante Tolvas · Comprender la tubería de automatización completa: Detectar → Activar → Recolectar → Almacenar.
1 · Resumen del Módulo — Integración y Escalada de Complejidad

Por qué el Módulo 4 es el "Módulo de Integración"

Cada módulo previo introdujo conceptos aislados. El Módulo 4 es el primer bloque del plan de estudios que exige explícitamente a los estudiantes combinar todas las habilidades anteriores dentro de un sistema unificado:

  • Del Módulo 1: Control espacial del Agente y el circuito base de Redstone de Observador y Pistón.
  • Del Módulo 2: Posicionamiento por coordenadas para el diseño de la cuadrícula de la granja.
  • Del Módulo 3: Estructura de lógica condicional ("si el cultivo está maduro, entonces cosechar").

El resultado final es un sistema agrícola completamente autónomo que opera de manera continua sin la intervención directa del jugador, constituyendo la construcción de mayor complejidad en el Nivel 1. Los estudiantes que completan con éxito este módulo demuestran la capacidad de diseñar, estructurar y depurar un sistema automatizado de múltiples componentes.

2 · El Agente como Jardinero — Siembra Automatizada

Programar al Agente para Sembrar en Filas

En el Módulo 1, los estudiantes desplazaban al Agente de forma manual con comandos de movimiento básicos. El Módulo 4 utiliza al Agente para sembrar semillas automáticamente en una cuadrícula estructurada, representando el primer uso del Agente como una herramienta de producción real y no como un objeto de demostración pasivo.

// Cargar primero las semillas en el inventario del Agente (clic derecho en el Agente, añadir semillas en la ranura 1)
on chat command "plant" {
  repeat 4 times { // siembra 5 semillas en una fila (bucle de 0 a 4)
    agent place on move down // coloca el elemento abajo a medida que el Agente avanza
    agent move forward 1 // avanza un bloque
  }
}

Condiciones críticas previas para el funcionamiento óptimo:

  1. Las semillas deben estar cargadas en el inventario del Agente antes de activar el comando de chat.
  2. El Agente debe estar posicionado en el punto inicial de la siembra, orientado hacia la dirección correcta de la fila.
  3. El suelo debe ser tierra de cultivo arada (usa una Azada sobre bloques de tierra o césped y colócales agua a una distancia máxima de 4 bloques).
  4. La siembra del Agente utiliza la instrucción "colocar al mover abajo", la cual sitúa el elemento en el bloque directamente inferior mientras realiza el avance.
⚠️
La falla de siembra más común del Agente: El Agente suelta las semillas en el suelo como entidades sueltas en lugar de plantarlas de forma fija. Esto ocurre si la superficie no es tierra de cultivo arada: las semillas colocadas en bloques de tierra normal son expulsadas del inventario sin convertirse en cultivos. Confirma siempre que la tierra esté arada e hidratada a menos de 4 bloques de una fuente de agua antes de iniciar el comando de siembra.

Siembra de Múltiples Filas — Bucles Anidados

Para estructurar un huerto completo de 5×5, los estudiantes necesitan utilizar bucles anidados: un bucle externo que administre las filas y un bucle interno que controle el avance y la siembra en cada columna. Esta es la introducción del Módulo 4 a la iteración anidada, un patrón lógico que prefigura el recorrido de matrices bidimensionales.

on chat command "farm" {
  repeat 5 times { // bucle externo: controla las 5 filas
    repeat 5 times { // bucle interno: controla las 5 columnas
      agent place on move down
      agent move forward 1
    }
    agent move right 1 // se desplaza lateralmente una fila a la derecha
    agent turn around // gira 180° para orientarse en la dirección de la siguiente fila
    agent move back 5 // regresa a la posición inicial de la fila
  }
}
3 · Circuitos Condicionales de Cosecha

El Circuito de Cosecha Observador–Pistón — Aplicado a Cultivos Tradicionales

Los estudiantes ya ensamblaron una cadena de Observador y Pistón para el bambú en el Módulo 1. El Módulo 4 traslada este mismo principio físico a cultivos como trigo, zanahorias o papas. La diferencia clave radica en la **detección de las etapas de crecimiento**:

  • Bambú: Crece en bloques completos definidos e independientes, facilitando su lectura directa.
  • Trigo/Zanahorias: Atraviesan 8 etapas secuenciales de crecimiento (del 0 al 7). **La etapa 7 representa la madurez total**. El Observador lee la transición de la etapa 6 a la 7 como un cambio de estado del bloque y emite el pulso instantáneo de energía.
  • Replica la estructura del Módulo 1: Cara del Observador apuntando hacia el bloque del cultivo maduro. Salida trasera del Observador hacia la entrada del Pistón o la línea de polvo de Redstone. El Pistón se despliega horizontalmente para romper el cultivo maduro de la fila.
💡
Enseñar el condicional físico "crecer para cosechar": Enmarca el comportamiento como una estructura condicional lógica de software, aunque esté construida con bloques físicos de Redstone: "SI el trigo alcanza la etapa 7 (madurez total), ENTONCES el Observador se activa, ENTONCES el Pistón se despliega y rompe el cultivo, ENTONCES los alimentos y semillas caen al suelo". Este enfoque conecta la lógica condicional abstracta de MakeCode del Módulo 3 con los circuitos de Redstone del Módulo 4.
4 · Mecánicas de las Tolvas y Sistemas de Clasificación de Elementos

¿Qué es una Tolva?

Una Tolva es un bloque con forma de embudo que ejerce dos funciones principales: (1) succiona de forma automática los elementos sueltos en el suelo que caen sobre su espacio superior, y (2) transfiere esos elementos desde su memoria interna hacia un contenedor adyacente (un cofre, un horno u otra Tolva) a través de su tubo de salida.

Reglas de Colocación de la Tolva

  • Dirección de salida: Determinada por el bloque sobre el cual se apoya al ser colocada: su tubo inferior apuntará directamente al contenedor objetivo.
  • Para conectar a un cofre: Mantén presionado Mayús (Shift) y haz clic derecho en el cofre. Sin presionar Mayús, se abrirá la interfaz del cofre en lugar de realizar el acoplamiento físico.
  • Para encadenar Tolvas: Conéctalas de extremo a extremo; los elementos se desplazarán por la cadena a una velocidad constante de 2.5 elementos por segundo por cada Tolva.
  • Capacidad interna: Cada Tolva cuenta con 5 ranuras de almacenamiento para grupos de objetos (hasta ×64 por grupo).

Sistema de Clasificación Base

Un mecanismo de separación simple utilizando dos cofres: se coloca una Tolva encima de cada cofre y se carga parcialmente con un objeto de filtro específico (el elemento que deseas enviar a ese contenedor en particular). Al circular el flujo de recolección, los objetos que coinciden con el filtro se almacenan en ese cofre, mientras que los elementos diferentes avanzan hacia la siguiente sección de la cadena.

Objetivo del Módulo 4: Enviar el trigo cosechado hacia un cofre de recolección y las semillas hacia un contenedor independiente, separando la producción alimenticia del material de siembra de forma automática.

Tiempos de Transferencia de la Tolva — Parámetros Prácticos

Tasa de transferencia técnica: una Tolva procesa exactamente un objeto cada 4 ticks de juego (0.2 segundos). A un ritmo estándar de 20 ticks por segundo, esto equivale a una velocidad de **2.5 elementos por segundo**. Para sistemas agrícolas a gran escala que generen una producción masiva, una sola línea de Tolvas puede crear un cuello de botella. Soluciones de optimización:

  • Desplegar cadenas de Tolvas en paralelo, multiplicando los puntos de succión hacia cofres independientes.
  • Utilizar una Vagoneta con tolva sobre rieles de recolección: las vagonetas con tolva succionan bloques completos a una velocidad dos veces mayor que las Tolvas fijas tradicionales.
5 · La Tubería de Automatización Completa

El Marco de Cuatro Etapas — Detectar → Activar → Recolectar → Almacenar

Este es el marco de enseñanza insignia del Módulo 4 de Kodland. Cualquier sistema agrícola automatizado complejo puede explicarse y modelarse mediante estas cuatro fases secuenciales e interdependientes:

Etapa 1: Detectar

El bloque Observador monitorea el bloque del cultivo leyendo de forma constante su estado biológico (etapa de crecimiento 6 → 7). Al detectar la madurez completa, emite un pulso instantáneo de Redstone de 1 tick.

Etapa 2: Activar

El pulso eléctrico del Observador viaja por el cableado y energiza al Pistón. El Pistón se despliega horizontalmente, rompiendo el bloque del cultivo maduro, el cual se transforma instantáneamente en objetos sueltos sobre el suelo.

Etapa 3: Recolectar

La Tolva, posicionada estratégicamente debajo de la zona de caída del cultivo, succiona los elementos sueltos. Los objetos ingresan a las 5 ranuras internas de la Tolva en un tiempo menor a 0.5 segundos tras entrar en su rango de succión.

Etapa 4: Almacenar

La Tolva transfiere los objetos recolectados hacia el Cofre acoplado a una velocidad de 2.5 elementos por segundo. El sistema de filtros se encarga de direccionar el trigo y las semillas hacia sus respectivos contenedores independientes.

✅
Procesamiento cognitivo profundo: Una vez que las cuatro etapas se encuentren construidas y operativas, solicita a los estudiantes describir el flujo de la tubería utilizando sus propias palabras y sin mirar las interfaces de código: "¿Qué componente detecta? ¿Qué activa? ¿Qué recolecta? ¿Qué almacena?". Esta evocación verbal consolida de forma definitiva el modelo analítico del sistema y los prepara para la fase de diseño de proyectos libres en el Módulo 5.
6 · Catálogo de Resolución de Problemas — Módulo 4 (5 Escenarios)
Error / SíntomaCausa RaízSolución ExactaGuión de Prevención
El Agente expulsa las semillas en el suelo en lugar de plantarlas de forma fija La superficie sobre la cual se desplaza el Agente es tierra normal o bloques de césped sin arar. Al colocar semillas en bloques no aptos para el cultivo, el software las expulsa como objetos sueltos. Antes de activar el comando de siembra: (1) Emplea una Azada sobre cada bloque de tierra para convertirlo en tierra de cultivo arada. (2) Instala agua a un máximo de 4 bloques de distancia para hidratar el suelo (marrón oscuro = hidratado, color ocre/claro = seco). (3) Verifica haciendo una prueba de avance inicial sobre un área arada. "Antes de que el Agente pueda sembrar, el suelo debe estar arado e hidratado. Verifica el color de la tierra: el marrón oscuro indica que el suelo está listo para recibir la semilla."
La Tolva se llena de objetos en su memoria pero no los transfiere hacia el cofre El tubo inferior de salida de la Tolva no está acoplado al bloque del cofre: se colocó la Tolva sin mantener presionado Mayús (Shift), abriendo la interfaz en su lugar. La salida apunta al aire en dirección hacia abajo. Rompe la Tolva y colócala de forma correcta: mantén presionado Mayús y haz clic derecho directamente sobre el bloque del cofre. Asegúrate visualmente de que el tubo inferior de salida apunte hacia el cofre. También puedes colocarla de forma lateral si el cofre se sitúa a un costado de la línea de recolección. "Recuerda mantener presionado Mayús al acoplar una Tolva a un cofre. Si no lo haces, abrirás el cofre y la Tolva quedará mal colocada."
El Observador parpadea al crecer el cultivo pero el Pistón no se despliega, interrumpiendo la cosecha La salida trasera del Observador no se encuentra alineada con el bloque del Pistón, o existe una interrupción física en el cable de polvo de Redstone. Es el mismo fallo de orientación del Módulo 1, pero se vuelve más complejo debido a la escala de la granja. Aplica la depuración en aislamiento: desconecta el Observador del resto del circuito. Instala una Lámpara de Redstone temporal en la salida trasera del Observador. Si la lámpara enciende al crecer el cultivo, el sensor opera bien y la falla se ubica adelante (un espacio vacío en el cable o mala orientación del Pistón). Corrige la línea o reubica el Pistón hacia la dirección del cultivo. "Prueba cada etapa por separado. ¿El Observador funciona? Entonces revisa el cable. ¿El cable funciona? Entonces verifica la orientación del Pistón."
El comando de siembra anidado del Agente planta bloques en un patrón en espiral o desordenado, en lugar de una cuadrícula perfecta La orientación final del Agente tras el comando de giro de 180° es incorrecta; el alumno utilizó la instrucción 'girar a la izquierda' dos veces pero no consideró que tras giros previos asimétricos, dos giros no siempre equivalen a un cambio de sentido exacto, o el conteo de bloques de retorno es incorrecto. Dibuja el camino de coordenadas del Agente en un papel antes de programar la secuencia. Registra cada giro de forma explícita. Verifica: al terminar la primera fila, ¿el Agente mira en la dirección opuesta? ¿El comando 'mover atrás 5' lo regresa al punto inicial del carril? Realiza pruebas con una sola fila antes de extender el bucle a 5. "Dibuja la ruta en papel antes de codificarla. ¿Hacia dónde mira el robot tras el primer giro? ¿Y tras el segundo? El robot no adivina el camino, solo ejecuta las instrucciones exactas del código."
El sistema de clasificación direcciona ABSOLUTAMENTE TODOS los objetos hacia el primer cofre, ignorando los filtros de separación Las ranuras internas de la Tolva de filtro no están configuradas correctamente: las ranuras deben estar parcialmente ocupadas con muestras del objeto para retener únicamente los elementos que coincidan. O no se colocó ningún elemento y la Tolva absorbe de manera indiscriminada. El crafteo de un clasificador por tolvas demanda colocar objetos de muestra en cada ranura de la Tolva de filtro sin dejar espacios vacíos. Los elementos coincidentes se acumulan en la Tolva, mientras que los diferentes avanzan en la línea. Método de simplificación para Nivel 1: usa Tolvas con filtros visuales directos y realiza pruebas manuales con dos objetos distintos (trigo frente a semillas) para verificar las rutas de almacenamiento. "La Tolva de filtro necesita una plantilla de muestra: coloca un elemento del objeto objetivo en sus ranuras. De esta forma retendrá los elementos que coincidan y dejará pasar los demás."
7 · Consejos Pro del Tutor y Analogías del Mundo Real

💡 Clasificación por Tolvas = La Analogía de la "Planta de Reciclaje"
"Piensa en una planta de reciclaje automatizada: diferentes bandas transportadoras dirigen las botellas de vidrio hacia un contenedor, el papel hacia otro y el metal hacia uno distinto. El sistema de Tolvas de tu granja es tu planta de reciclaje digital. El trigo se mueve a la izquierda y las semillas a la derecha, de forma automática y constante, sin que nadie tenga que separarlos a mano".

💡 La Tubería de Cuatro Etapas = La Analogía del "Centro de Distribución de Amazon"
"Cuando haces un pedido en línea: (1) Un sensor digital detecta tu compra [Observador]. (2) Un brazo robótico toma el producto del estante de almacenamiento [Pistón]. (3) Una banda transportadora desplaza el paquete hacia la zona de embalaje [Tolva]. (4) El paquete se guarda en un contenedor etiquetado en la sección correcta del almacén [Cofre]. Nuestra granja funciona con las mismas cuatro etapas, pero procesando trigo en lugar de cajas".

💡 Bucles Anidados = La Analogía de la "Impresora de Escritorio" para Cuadrículas 2D
"Cuando una impresora imprime una página de texto, el cabezal se desplaza horizontalmente imprimiendo una fila de puntos (bucle interno), luego baja un renglón (bucle externo) y avanza imprimiendo la siguiente fila, repitiendo el proceso hasta completar la hoja. La siembra por filas del Agente opera de forma idéntica: el bucle interno siembra una fila de semillas y el bucle externo realiza el cambio de fila".

💡 Pruebas en Aislamiento — La Regla de "Depurar una Etapa a la Vez"
Para sistemas integrados de múltiples componentes, realiza siempre pruebas aisladas en cada etapa antes de unirlas en el circuito final. El protocolo de depuración para el Módulo 4 establece: (1) Probar que el Observador emita el pulso al crecer el cultivo (usa una lámpara de Redstone). (2) Probar que el Pistón se despliegue de forma manual (usa una Palanca). (3) Probar que la Tolva succione objetos (tira elementos con la mano sobre ella). (4) Probar que la Tolva envíe los elementos al cofre correcto (coloca objetos de prueba en el filtro). Solo cuando cada fase pase su prueba individual, los estudiantes deben conectar la tubería completa.
