Teoria Techniczna — Trójwymiarowy Układ Współrzędnych (X, Y, Z)

Definicje osi i układ odniesienia świata

Minecraft używa prawoskrętnego układu współrzędnych kartezjańskich 3D. Każdy blok w nieskończonym świecie zajmuje unikalną pozycję określoną przez trzy liczby całkowite ze znakiem:

Oś X → Wschód (+X) / Zachód (−X) ← pozioma pozycja wschód-zachód
Oś Y → Góra (+Y) / Dół (−Y) ← WYSOKOŚĆ — wysokość nad skałą macierzystą (bedrock)
Oś Z → Południe (+Z) / Północ (−Z) ← pozioma pozycja północ-południe

Warstwa skały macierzystej (bedrock) = Y 0 (niezniszczalna w Survivalu)
Poziom morza = Y 63 (domyślny płaski świat Overworld)
Limit budowania w górę = Y 255 (starsze edycje) / Y 320 (1.18+)

Jak odczytać bieżącą pozycję:
· Java Edition: naciśnij F3 — XYZ wyświetlane w lewym górnym rogu
· Minecraft Education Edition: Ustawienia → Świat → włącz „Pokaż współrzędne” w HUD

Współrzędne Bezwzględne — Stałe Adresy w Świecie

// Bezwzględne — teleporuje do stałego adresu świata niezależnie od bieżącej pozycji
/tp @s 100 64 200 → siebie do X=100, Y=64, Z=200 (zawsze)
/tp @a 0 64 0 → wszystkich dokładnie do środka (początku) świata
/tp @a [X] [Y] [Z] → wszystkich do dzisiejszych współrzędnych spawnu (przypnij na czacie)

Współrzędne Względne — Prefiks Tildy ~

Tilda oznacza „przesunięcie w stosunku do mojej bieżącej pozycji w tym konkretnym momencie”. Jest obliczana dynamicznie w momencie uruchomienia polecenia — nie jest to stały adres. Przesunięcia względne mogą być dodatnie lub ujemne:

// ~ = "stamtąd, gdzie jestem teraz"
/tp @s ~0 ~0 ~0 → przesunięcie zero = brak ruchu (zostaje w miejscu)
/tp @s ~3 ~0 ~0 → 3 bloki na Wschód od bieżącej pozycji X
/tp @s ~0 ~5 ~0 → 5 bloków prosto w GÓRĘ
/tp @s ~0 ~-3 ~0 → 3 bloki w DÓŁ

// Składnia mieszana — prawidłowa i często przydatna:
/tp @s ~0 64 ~0 → zachowaj bieżące X i Z; ustaw wysokość dokładnie na Y=64
/tp @s 100 ~0 ~0 → ustaw X=100; zachowaj bieżące Y i Z

POWSZECHNY BŁĄD: „/tp @a ~0 ~0 ~0 przenosi wszystkich graczy w to samo miejsce”
PRAWIDŁOWO: „~0 ~0 ~0 = zerowe przesunięcie względem WŁASNEJ pozycji KAŻDEGO gracza = nikt się nie rusza”
💡
Fizyczne nauczanie różnicy: „Współrzędne bezwzględne są jak adres zamieszkania — ulica Główna 100 to zawsze ulica Główna 100. Współrzędne względne są jak wskazówki dojścia — '3 bloki w lewo' zależy od tego, w którą stronę aktualnie się obracasz”.
Teoria Techniczna — Interfejs nakładki MakeCode

Dostęp do MakeCode z poziomu Minecrafta

Ścieżka dostępu: Naciśnij T, aby otworzyć czat Minecrafta → wpisz /code → naciśnij Enter. Edytor MakeCode otworzi się jako pełnoekranowa nakładka. Kod pozostaje zapisany między otwarciami — zamknięcie i ponowne otwarcie pokazuje ten sam stan programu.

Krytyczny warunek wstępny: Opcja „Włącz Code Builder” (Kreator kodu) musi być włączona (ON) w ustawieniach świata (ikona ołówka na miniaturze świata na liście światów), zanim jakikolwiek uczeń wejdzie do świata. Jeśli ta opcja jest wyłączona (OFF), komenda /code uruchomi się po cichu, nie dając żadnej reakcji — uczniowie nic nie zobaczą i nie pojawi się żaden komunikat o błędzie. Jest to najczęstszy błąd przed sesją.

Strefy Interfejsu MakeCode — Co Uczniowie Muszą Wiedzieć w Module 1

Panel Lewy — Paleta Bloków

Sparametryzowana biblioteka bloków. Kategorie używane w Module 1: Pętle (Repeat), Agent (przemieszczaj, obracaj, umieszczaj), Gracz (Przy poleceniu czatu). Bloki są przeciągane na płótno. Kliknięcie prawym przyciskiem myszy na umieszczony blok otwiera opcje zmiany nazwy/usunięcia.

Panel Prawy — Płótno Kodowania

Gdzie bloki są montowane. Płótno jest nieskończone — powiększaj za pomocą kółka myszy. Bloki muszą być połączone (zatrzaśnięte razem), aby działać; pływające bloki nic nie wykonują. Zielony przycisk „play” uruchamia program. Czerwony przycisk „stop” zatrzymuje wszystkie działające pętle.

Podstawowe typy bloków — Wiedza wymagana w Module 1

// 1. PRZY POLECENIU CZATU — Rejestrator zdarzeń. Uruchamia wewnętrzny kod, gdy gracz wpisze dane słowo na czacie MC.
on chat command "build" {
  // Kod tutaj działa TYLKO wtedy, gdy gracz wpisze "build" na czacie Minecrafta
  // Kod POZA tym blokiem uruchamia się tylko raz przy ładowaniu programu
}

// 2. PĘTLA POWTÓRZEŃ — Wykonuje zamknięty kod dokładnie N razy sekwencyjnie.
repeat 5 times {
  agent move forward 1 // działa łącznie 5 razy
}

// 3. RUCH AGENTA — Przesuwa Agenta o N kroków w kierunku względnym do AKTUALNEGO ZWROTU Agenta.
agent move forward 3 // 3 bloki w kierunku przednim Agenta
agent move back 1 // 1 blok za Agentem
agent move left 2 // 2 bloki po lewej stronie Agenta (względem zwrotu)
agent turn left // obraca Agenta o 90° w lewo — zmienia WSZYSTKIE kolejne kierunki

Mechanika Agenta — Orientacja, Ekwipunek i Najbardziej Krytyczna Koncepcja

Każde polecenie ruchu Agenta jest oceniane względem aktualnego kierunku zwrotu Agenta — a nie kompasu świata. Zwrot ten zmienia się po każdym poleceniu Obróć. Uczniowie, którzy nie będą tego wyraźnie kontrolować, za każdym razem uzyskają ukośne lub błędne ścieżki.

// Agent zaczyna zwrócony na PÓŁNOC (kierunek −Z)
agent move forward 3 → przemieszcza się na PÓŁNOC o 3 bloki (Z maleje o 3)
agent turn right → Agent jest teraz zwrócony na WSCHÓD (+X)
agent move forward 2 → przemieszcza się na WSCHÓD o 2 bloki (już NIE na północ!)
agent turn right → Agent jest teraz zwrócony na POŁUDNIE (+Z)
agent move forward 1 → przemieszcza się na POŁUDNIE o 1 blok

ZASADA: Po każdym bloku Obróć, "do przodu" oznacza inny kierunek świata.
Wyraźnie kontroluj zwrot. Rysuj na papierze przed zakodowaniem sekwencji z wieloma zwrotami.
⚠️
Wymóg przed kodowaniem: Zanim jakikolwiek uczeń napisze kod ruchu Agenta ze zwrotami, poproś go o narysowanie mapy od góry pokazującej: (a) pozycję startową Agenta, (b) początkową strzałkę zwrotu, (c) każdy zwrot ze zaktualizowaną strzałką. Uczniowie, którzy najpierw rysują, piszą kod poprawnie w ~75% przypadków; ci, którzy kodują bezpośrednio, generują błędy w ~65% przypadków.

Ekwipunek Agenta

Agent posiada ekwipunek składający się z 27 slotów (siatka 3×9), identyczny pod względem pojemności ze skrzynią. Przedmioty muszą być załadowane do Agenta, zanim będzie mógł je umieścić. Ładowanie: stań obok Agenta i kliknij na niego prawym przyciskiem myszy, aby otworzyć interfejs ekwipunku, a następnie przeciągnij przedmioty ze swojego ekwipunku lub pobliskiej skrzyni.

// Agent umieszcza blok — wymaga wcześniejszego załadowania przedmiotów do ekwipunku Agenta
agent place on move down // umieszcza przedmiot pod Agentem podczas ruchu (do obsiewania rzędów)
agent place forward // umieszcza przedmiot 1 blok przed Agentem
agent place down // umieszcza przedmiot bezpośrednio pod bieżącą pozycją

// Typowy błąd: Agent nic nie robi, gdy wywoływane jest polecenie umieszczenia.
// Przyczyna: slot ekwipunku jest pusty. Zawsze ładuj przed uruchomieniem poleceń umieszczania.
Teoria Techniczna — Fizyka Sygnału Redstone

Siła Sygnału, Propagacja i Spadek na Dystansie 15 Bloków

Każde źródło zasilania Redstone emituje sygnał o sile 15. Każdy blok proszku Redstone, przez który przechodzi sygnał, zmniejsza jego siłę o dokładnie 1. Przy sile 0 sygnał wygasa — obwód zostaje przerwany.

// Siła sygnału w zależności od odległości (bez Repeatera)
Proszek[1]: siła=15 → Proszek[5]: siła=11 → Proszek[10]: siła=6
→ Proszek[14]: siła=2 → Proszek[15]: siła=1 → Proszek[16]: siła=0 → WYGASŁY

Maksymalna długość przewodu przed Repeaterem: 15 bloków

Rozwiązanie: Umieść Repeater Redstone na 15. bloku lub wcześniej.
Repeater resetuje siłę sygnału do poziomu 15, niezależnie od siły sygnału wejściowego.
Repeatery dodają również konfigurowalne opóźnienie: 1–4 ticki (1 tick = 0.05 sekundy).

Cztery Komponenty, Które Uczniowie Muszą Znać w Module 1

Blok Obserwatora

Wykrywa zmiany stanu bloku w pojedynczym bloku, na który bezpośrednio patrzy jego twarz (strona z wyrzeźbioną teksturą). Emituje impuls Redstone o długości 1 ticka, gdy wykryje zmianę stanu. W przypadku bambusa: wzrost z jednej fazy do drugiej to zmiana stanu bloku. Obserwator nie emituje sygnału ciągłego — jedynie krótki impuls.

Dwie strony do zapamiętania: Twarz (wyrzeźbiona tekstura) musi być skierowana na bambus. Tył (wyjście, gładka strona) musi być skierowany na Tłok lub przewód łączący. Pomylenie którejkolwiek z tych stron przerywa obwód po cichu.

Tłok (Zwykły)

Wysuwa się o 1 blok po podłączeniu zasilania; cofa się czysto po odłączeniu zasilania. Używany do łamania bambusa: głowica Tłoka wysuwa się w blok bambusa, łamiąc go i powodując spadanie przedmiotów jako encje.

Zwykły a Lepki: Lepki Tłok przyciąga sąsiedni blok z powrotem po odłączeniu zasilania. Do zbioru bambusa używaj zwykłego Tłoka — łamie bambus przy wysunięciu i cofa się pozostawiając wolną przestrzeń, gotową do kolejnego cyklu wzrostu.

Przycisk a Dźwignia

Przycisk (Kamienny): Chwilowy impuls o długości ~0.5 sekundy, a następnie automatycznie powraca do stanu off. Idealny do aktywacji jednorazowych. Dźwignia: Ciągły sygnał, dopóki nie zostanie ręcznie przełączona z powrotem. Idealna dla stanów, które muszą pozostać włączone/wyłączone (np. brama, która ma pozostać otwarta).

Znajomość tego rozróżnienia zapobiega typowemu błędowi polegającemu na użyciu Przycisku tam, gdzie wymagany jest sygnał ciągły — np. próba utrzymania włączonej Lampy Redstone za pomocą Przycisku (emituje impuls i gaśnie).

Repeater Redstone

Resetuje sygnał z powrotem do siły 15. Dodaje opóźnienie 1–4 ticki (kliknij prawym przyciskiem myszy, aby przewijać wartości). Kierunkowy: musi być umieszczony w linii ze ścieżką sygnału, a nie prostopadle. Typowy błąd: Repeater umieszczony w poprzek przewodu pod kątem 90° — nie odbiera sygnału i nic nie transmituje, ale ten błąd jest wizualnie niewidoczny.

Teoria Techniczna — Automatyczna Farma Bambusa (Schemat Budowy)

Mapa komponentów przed budową

// Widok przekroju pionowego — B = blok, · = powietrze

Y+3 · · ·
Y+2 · · · ← Tłok (skierowany w DÓŁ, zasilany przez Obserwator)
Y+1 B · O ← B = blok bambusa Y+1, O = Obserwator (twarz→bambus, tył→Tłok)
Y+0 B H · ← B = korzeń bambusa, H = Lej (połączony ze Skrzynią obok)
Y-1 · C · ← C = Skrzynia (odbiera przedmioty z Leja)

Przewód: Wyjście z tyłu Obserwatora → proszek Redstone → wejście Tłoka (jeśli nie leżą bezpośrednio obok siebie)

Sekwencja Budowy Krok po Kroku

  1. Posadź bambus na bloku trawy lub ziemi. Upewnij się, że masz ≥ 3 bloki wolnej przestrzeni w pionie powyżej — bambus rośnie po 1 bloku na raz, maksymalnie do 16 bloków.
  2. Umieść Tłok bezpośrednio nad blokiem bambusa na wysokości Y+2, skierowany w dół. Po aktywacji wysunie się w dół w bambus na wysokości 2, łamiąc go.
  3. Umieść Obserwator obok łodygi bambusa na wysokości Y+1. Twarz (wyrzeźbiona tekstura) musi wskazywać na blok bambusa. Tył (gładka strona) musi wskazywać na Tłok — lub na łączący go proszek Redstone.
  4. Połącz Obserwator z Tłokiem za pomocą proszku Redstone, jeśli nie są bezpośrednio sąsiadujące. Pamiętaj o maksimum 15 bloków — dodaj Repeater, jeśli przewód jest dłuższy.
  5. Zweryfikuj orientację: Kliknij prawym przyciskiem myszy na Obserwator i sprawdź, czy jego wskaźnik wyjściowy pokrywa się z przewodem. Częsta metoda kontroli: umieść tymczasową Lampę Redstone na przewodzie i potwierdź, że wysyła impuls, gdy bambus rośnie.
  6. Dodaj zbieranie lejem: Przytrzymaj Shift i kliknij prawym przyciskiem myszy na blok pod korzeniem bambusa, aby umieścić Lej skierowany do Skrzyni z boku. Lej automatycznie zbiera upadłe przedmioty i deponuje je w Skrzyni.
  7. Test: Poczekaj, aż bambus urośnie o jedną fazę. Sekwencja: bambus rośnie → Obserwator wykrywa zmianę stanu → emituje impuls 1-tickowy → Tłok się wysuwa → bambus na wysokości 2 łamie się → spadają przedmioty → Lej je zbiera → przedmioty pojawiają się w Skrzyni.
✅
Kamień milowy sukcesu: Przedmioty gromadzą się w Skrzyni bez żadnego działania ze strony gracza. To pierwsze doświadczenie uczniów z prawdziwą autonomiczną automatyzacją. Świętuj je wyraźnie za pomocą konkretnej pochwały powiązanej z zachowaniem.
Teoria Techniczna — Generator Bruku z Wody + Lawy

Chemia Kolizji — Co Dokładnie Się Dzieje i Dlaczego

// Trzy możliwe interakcje Woda × Lawa:

Źródło Wody + płynąca Lawa → Bruk (Cobblestone) ← CEL MODUŁU 1
Źródło Lawy + płynąca Woda → Obsydian ← trwały; wymaga Diamentowego Kilofa
Źródło Lawy + źródło Wody → brak nowego bloku ← tylko granica płynów; brak konwersji

KLUCZOWA ZASADA: Dla uzyskania Bruku LAWA musi być płynąca, a nie stanowić bloku źródła.
Jeden pusty blok odstępu między blokami źródłowymi zmusza oba płyny do przepływu przed spotkaniem.

Układ Budowy

  1. Wykop kanał o szerokości 3 bloków i głębokości 1 bloku w dowolnym płaskim obszarze.
  2. Umieść blok źródła Wody na skrajnym lewym końcu kanału (wiadro wody, kliknij prawym przyciskiem myszy na podłogę).
  3. Umieść blok źródła Lawy na skrajnym prawym końcu. Pomiędzy dwoma blokami źródłowymi musi być dokładnie 1 pusty blok odstępu — to nie podlega negocjacjom.
  4. Woda płynie w prawo; Lawa płynie w lewo. Spotykają się w środkowym odstępie, a płynąca Lawa krzepnie w kontakcie z przepływem Wody, wytwarzając Bruk.
  5. Wydobądź blok Bruku za pomocą dowolnego Kilofa. Oba przepływy natychmiast spotykają się ponownie w opuszczonym odstępie i generują nowy blok Bruku. Ten cykl jest nieskończony.
⚠️
Błąd z Obsydianem: Uczniowie umieszczają blok źródła Lawy bezpośrednio obok źródła Wody (odstęp zero). Produkuje to Obsydian zamiast Bruku, ponieważ teraz oba stacjonarne źródła stykają się bezpośrednio. Obsydian wymaga Diamentowego Kilofa do wydobycia i nie można go rozbić na etapie Modułu 1. Zawsze egzekwuj zasadę 1 bloku odstępu przed umieszczeniem Lawy.

Analogie Pojęciowe — Uproszczenie Abstrakcji dla Wieku 8–9 Lat

💡 Redstone = „Domowa Instalacja Elektryczna”
„Proszek Redstone to przewód wewnątrz ścian Twojego domu. Dźwignia lub przycisk to włącznik światła. Tłok to silnik lub żarówka. I tak jak prawdziwy prąd może przemieszczać się tylko na określoną odległość, zanim napięcie spadnie zbyt nisko, sygnał Redstone wygasa po 15 blokach — dlatego potrzebujemy Repeatera, który działa jak wzmacniacz w długim kablu głośnikowym”.

💡 Obserwator = „Powiadomienie SMS z Czujnika Ruchu w Ogrodzie”
„Wyobraź sobie, że posadziłeś słonecznik i chciałeś wiedzieć w tej samej sekundzie, kiedy urósł wyższy. Mógłbyś gapić się na niego cały dzień — albo postawić obok niego czujnik ruchu, który wyśle Ci wiadomość na telefon, gdy tylko się poruszy. Obserwator to ten czujnik. Gdy tylko bambus urośnie choćby o milimetr, wysyła wiadomość SMS za pomocą Redstone do Tłoka”.

💡 Współrzędne = „System Adresów Ulicznych w Mieście”
„Każdy budynek w mieście ma swój adres: ulica Wschodnia 100, Piąta Aleja, Piętro 3. W Minecraft X to numer Twojej ulicy na linii Wschód-Zachód, Z to numer Twojej alei na linii Północ-Południe, a Y to piętro, na którym się znajdujesz. Twoja farma bambusa ma swój dokładny adres — X=100, Y=64, Z=200 — i żaden inny blok w całym nieskończonym świecie nie dzieli z nią tego adresu”.

💡 Pętla Powtórzeń = „Kserokopiarka”
„Napisałeś idealne zdanie na kartce i potrzebujesz 20 kopii. Mógłbyś zapisać je ręcznie 20 razy — albo włożyć do kserokopiarki i wpisać '20'. Blok Powtórz to Twoja kserokopiarka. Zapisujesz akcję raz, mówisz, ile kopii potrzebujesz, a ona wykonuje je wszystkie, podczas gdy Ty odpoczywasz”.

💡 Agent = „Robot, Który Wykonuje Tylko Pisemne Instrukcje”
„Agent to Twój robotyczny najlepszy przyjaciel — ale jest jedna rzecz: potrafi robić tylko to, co dla niego wcześniej zapiszesz. Nie potrafi improwizować. Jeśli powiesz 'idź do przodu', pójdzie do przodu. Jeśli powiesz 'skręć w prawo, idź do przodu', skręci w prawo, a potem pójdzie do przodu. Wykona Twoje instrukcje idealnie, nawet jeśli te instrukcje zrzucą go z klifu. Dlatego planujemy, zanim zakodujemy”. (Przedstaw go najpierw jako postać: „Czy to dinozaur? Kurczak? Nie — to Twój nowy najlepszy przyjaciel: Agent!”)

💡 Współrzędne Względne a Bezwzględne — „GPS a Wskazówki Osobiste”
„Współrzędne bezwzględne są jak GPS — ulica Główna 100 to zawsze ulica Główna 100, niezależnie od tego, gdzie się znajdujesz. Współrzędne względne są jak udzielanie komuś wskazówek z miejsca, w którym aktualnie stoi — 'skręć w lewo i przejdź 3 bloki'. Tilda (~) oznacza 'stamtąd, gdzie stoję w tym momencie'. Dlatego dwóch różnych graczy używających /tp @s ~3 ~0 ~0 wyląduje w dwóch różnych miejscach — obaj przesunęli się o 3 bloki na wschód od swoich odpowiednich pozycji”.
