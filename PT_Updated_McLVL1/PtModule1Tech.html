Módulo 1: Admirável Mundo da Pedra Verde — a referência técnica completa para tudo o que você precisa ensinar, explicar, demonstrar e solucionar problemas. Apenas mecânica pura; nenhuma pedagogia reside aqui.

Visão Geral do Módulo 1 — Escopo & Contexto dos Pré-requisitos

O que os Alunos Devem Dominar Após o Módulo 1

O Módulo 1 é o ponto de entrada de toda a espiral do Nível 1. Cada conceito introduzido aqui é um pré-requisito direto para os módulos subsequentes — coordenadas para o Módulo 2, loops do MakeCode para os Módulos 3 e 4, lógica de Redstone para o Módulo 4. Os alunos saem do Módulo 1 capazes de: navegar em 3D sem se perder, invocar e comandar o Agente, entender a Redstone como um sistema de sinal e demonstrar uma construção automatizada funcional. Esses não são resultados aspiracionais — são requisitos estruturais para que o Módulo 2 funcione.

Verificações Técnicas Pré-Sessão Específicas do Módulo 1

Antes do início da sessão, verifique estes itens específicos do Módulo 1 (além do Pré-Voo padrão da página de Pedagogia):

  1. Abra o mundo da sessão e digite /code no chat. Se a interface do MakeCode não aparecer, vá para as configurações do mundo e habilite o "Code Builder" (Criador de Código) antes de os alunos entrarem.
  2. Confirme se a zona de construção da fazenda de bambu possui pelo menos 3×3 blocos de terreno limpos e 5 blocos de espaço vertical livre. Pré-construa um Observador e um Pistão como referência de demonstração.
  3. Prepare uma vala de 3 blocos de largura para a demonstração da fábrica de pedregulho. Pré-posicione a fonte de Água em uma das extremidades para que os alunos precisem apenas colocar a Lava.
  4. Fixe as coordenadas de spawn de hoje na sua própria janela de chat e no chat do Zoom/Meet antes que os alunos cheguem.

Teoria Técnica — Sistema de Coordenadas 3D (X, Y, Z)

Definições de Eixo & Sistema de Referência do Mundo

O Minecraft usa um sistema de coordenadas cartesianas 3D dextrógiro. Cada bloco no mundo infinito ocupa uma posição única definida por três inteiros com sinal:

Eixo X → Leste (+X) / Oeste (−X) ← posição horizontal leste-oeste
Eixo Y → Para cima (+Y) / Para baixo (−Y) ← ALTITUDE — altura acima da rocha matriz
Eixo Z → Sul (+Z) / Norte (−Z) ← posição horizontal norte-sul

Camada de rocha matriz (bedrock) = Y 0 (indestrutível no Sobrevivência)
Nível do mar = Y 63 (padrão do mundo plano do Overworld)
Limite de construção no céu = Y 255 (edições antigas) / Y 320 (1.18+)

Como ler a posição atual:
· Java Edition: pressione F3 — XYZ exibido no canto superior esquerdo
· Minecraft Education Edition: Configurações → Mundo → habilite "Mostrar Coordenadas" no HUD

Coordenadas Absolutas — Endereços Fixos no Mundo

// Absolutas — teletransporta para um endereço fixo do mundo, independentemente da posição atual
/tp @s 100 64 200 → você mesmo para X=100, Y=64, Z=200 (sempre)
/tp @a 0 64 0 → todos para a origem do mundo exatamente
/tp @a [X] [Y] [Z] → todos para as coordenadas de spawn de hoje (fixe no chat)

Coordenadas Relativas — O Prefixo de Til ~

O til significa "deslocamento a partir da minha posição atual agora mesmo." Ele é avaliado dinamicamente no momento em que o comando é executado — não é um endereço fixo. Os deslocamentos relativos podem ser positivos ou negativos:

// ~ = "a partir de onde estou agora"
/tp @s ~0 ~0 ~0 → deslocamento zero = nenhum movimento (permanece no lugar)
/tp @s ~3 ~0 ~0 → 3 blocos a Leste da posição X atual
/tp @s ~0 ~5 ~0 → 5 blocos para cima direto
/tp @s ~0 ~-3 ~0 → 3 blocos para BAIXO

// Sintaxe mista — válida e frequentemente útil:
/tp @s ~0 64 ~0 → mantém o X e Z atuais; ajusta a altitude para Y=64 exatamente
/tp @s 100 ~0 ~0 → ajusta para X=100; mantém o Y e Z atuais

EQUÍVOCO COMUM: "/tp @a ~0 ~0 ~0 move todos os jogadores para o mesmo ponto."
CORRETO: "~0 ~0 ~0 = zero deslocamento a partir da posição de CADA jogador = ninguém se move."
💡
Ensinando a diferença fisicamente: "Coordenadas absolutas são como um endereço de rua — a Rua Principal, 100 sempre será a Rua Principal, 100. Coordenadas relativas são como direções de caminhada — 'ande 3 blocos à sua esquerda' depende de para onde você está virado no momento."
Teoria Técnica — Interface Sobreposta do MakeCode

Acessando o MakeCode de Dentro do Minecraft

Caminho de acesso: Pressione T para abrir o chat do Minecraft → digite /code → pressione Enter. O editor do MakeCode se abre como uma sobreposição em tela cheia. O código é persistente entre as aberturas — fechar e reabrir mostra o mesmo estado do programa.

Pré-condição crítica: "Enable Code Builder" (Habilitar Criador de Código) deve estar ATIVADO nas configurações do mundo (o ícone de lápis na miniatura do mundo na lista de Mundos) antes que qualquer aluno entre no mundo. Se essa opção estiver DESATIVADA, o comando /code roda silenciosamente sem resposta — os alunos não veem nada e nenhuma mensagem de erro aparece. Esta é a falha mais comum antes do início da sessão.

Zonas da Interface do MakeCode — O que os Alunos Precisam Saber no Módulo 1

Painel Esquerdo — Paleta de Blocos

Biblioteca de blocos categorizada. Categorias do Módulo 1 utilizadas: Loops (Repetir), Agente (mover, virar, colocar), Jogador (ao comando de chat). Os blocos são arrastados para a tela de código. Clicar com o botão direito em um bloco colocado revela opções para renomear/excluir.

Painel Direito — Tela de Código

Onde os blocos são montados. A tela é infinita — faça zoom com a roda do mouse. Os blocos devem estar conectados (encaixados uns nos outros) para funcionar; blocos soltos não executam nada. O botão verde de "play" roda o programa. O botão vermelho de "stop" interrompe todos os loops em execução.

Tipos de Blocos Centrais — Conhecimento Obrigatório do Módulo 1

// 1. AO COMANDO DE CHAT — Manipulador de eventos. Roda o código inserido quando o jogador digita a palavra no chat do MC.
on chat command "build" {
  // O código aqui roda APENAS quando o jogador digita "build" no chat do Minecraft
  // O código FORA deste bloco roda apenas uma vez no carregamento do programa
}

// 2. LOOP DE REPETIÇÃO — Executa o código inserido exatamente N vezes sequencialmente.
repeat 5 times {
  agent move forward 1 // roda 5 vezes no total
}

// 3. MOVER AGENTE — Move o Agente N passos em uma direção relativa ao DIRECIONAMENTO ATUAL do Agente.
agent move forward 3 // 3 blocos na direção frontal do Agente
agent move back 1 // 1 bloco atrás do Agente
agent move left 2 // 2 blocos à esquerda do Agente (relativo ao direcionamento)
agent turn left // rotaciona o Agente 90° no sentido anti-horário — altera TODOS os direcionamentos subsequentes

Mecânicas do Agente — Orientação, Inventário e o Conceito Mais Crítico

Cada comando de movimento do Agente é avaliado em relação à **direção atual para a qual o Agente está virado** — não em relação à bússola do mundo. Esse direcionamento muda após cada comando de Virar. Os alunos que não rastrearem isso explicitamente produzirão caminhos diagonais ou incorretos todas as vezes.

// O Agente começa virado para o NORTE (direção −Z)
agent move forward 3 → move-se para o NORTE 3 blocos (Z diminui em 3)
agent turn right → O Agente agora está virado para o LESTE (+X)
agent move forward 2 → move-se para o LESTE 2 blocos (NÃO para o norte mais!)
agent turn right → O Agente agora está virado para o SUL (+Z)
agent move forward 1 → move-se para o SUL 1 bloco

REGRA: Após cada bloco de Virar, "para frente" significa uma direção diferente no mundo.
Rastreie o direcionamento explicitamente. Desenhe no papel antes de programar qualquer sequência com várias curvas.
⚠️
Requisito pré-programação: Antes que qualquer aluno escreva um código de movimento do Agente com curvas, peça para ele desenhar um mapa de cima para baixo mostrando: (a) a posição inicial do Agente, (b) a seta de direcionamento inicial, (c) cada curva com uma seta atualizada. Alunos que desenham primeiro programam corretamente ~75% das vezes; aqueles que programam direto produzem erros ~65% das vezes.

Inventário do Agente

O Agente possui um inventário de 27 slots (grade 3×9), idêntico em capacidade a um baú. Os itens devem ser carregados no Agente antes que ele possa colocá-los. Carregando: fique ao lado do Agente e clique com o botão direito nele para abrir a interface gráfica do inventário, depois arraste os itens do seu próprio inventário ou de um baú próximo.

// Agente coloca bloco — requer itens pré-carregados no inventário do Agente
agent place on move down // coloca o item abaixo do Agente enquanto ele se move (para fileiras de cultivo)
agent place forward // coloca o item 1 bloco na frente do Agente
agent place down // coloca o item diretamente abaixo da posição atual

// Erro comum: O Agente não faz nada quando o comando de colocar é chamado.
// Causa: o slot do inventário está vazio. Sempre carregue itens antes de rodar os comandos de colocar.
Teoria Técnica — Física do Sinal de Redstone

Força do Sinal, Propagação e Decaimento de 15 Blocos

Cada fonte de energia de Redstone emite um sinal com **força 15**. Cada bloco de pó de Redstone pelo qual o sinal passa decrementa a força em exatamente 1. Na força 0, o sinal morre — o circuito é interrompido.

// Força do sinal vs. distância (sem Repetidor)
Pó[1]: força=15 → Pó[5]: força=11 → Pó[10]: força=6
→ Pó[14]: força=2 → Pó[15]: força=1 → Pó[16]: força=0 → MORTO

Comprimento máximo do fio antes do Repetidor: 15 blocos

Solução: Coloque um Repetidor de Redstone no bloco 15 ou antes dele.
O Repetidor redefine a força do sinal para 15, independentemente da força de entrada.
Os repetidores também adicionam um atraso configurável: 1 a 4 ticks (1 tick = 0,05 segundos).

Os Quatro Componentes que os Alunos Devem Conhecer no Módulo 1

Bloco do Observador

Detecta *mudanças de estado do bloco* no único bloco para o qual sua **face** (lado da textura esculpida) aponta diretamente. Emite um pulso de Redstone de 1 tick quando uma mudança de estado é detectada. Para o bambu: crescer de um estágio para o outro é uma mudança de estado do bloco. O Observador não emite um sinal contínuo — apenas um breve pulso.

Dois lados que devem ser conhecidos: A face (textura esculpida) deve apontar para o bambu. A parte traseira (saída, lado liso) deve apontar para o Pistão ou para o fio de conexão. Errar qualquer um dos lados interrompe o circuito silenciosamente.

Pistão (Normal)

Estende-se por 1 bloco quando alimentado; retrai-se de forma limpa quando perde a energia. Usado para quebrar o bambu: a cabeça do Pistão se estende para dentro do bloco de bambu, quebrando-o e fazendo com que os itens caiam como entidades.

Normal vs. Grudento: Um Pistão Grudento puxa o bloco adjacente de volta quando perde a energia. Para a colheita de bambu, use um Pistão *normal* — ele quebra o bambu ao se estender e se retrai de forma limpa, pronto para o próximo ciclo de crescimento.

Botão vs. Alavanca

Botão (Pedra): Pulso momentâneo de ~0,5 segundos, depois retorna automaticamente para desligado. Ideal para ativações de disparo único. Alavanca: Sinal contínuo até ser alternada manualmente de volta. Ideal para estados que devem permanecer ligados/desligados (por exemplo, um portão que deve ficar aberto).

Conhecer essa distinção evita o erro comum de usar um Botão onde um sinal contínuo é necessário — por exemplo, tentar manter uma Lâmpada de Redstone acesa com um Botão (ela pulsa e apaga).

Repetidor de Redstone

Redefine o sinal de volta para a força 15. Adiciona um atraso de 1 a 4 ticks (clique com o botão direito para alternar). Direcional: deve ser colocado *em linha* com o caminho do sinal, não perpendicular. Erro comum: Repetidor colocado transversalmente ao fio a 90° — ele não recebe sinal e não transmite nada, mas essa falha é visualmente imperceptível.

Teoria Técnica — Fazenda de Bambu Automática (Projeto de Construção)

Mapa de Componentes Antes da Construção

// Visão de seção transversal vertical — B = bloco, · = ar

Y+3 · · ·
Y+2 · · · ← Pistão (voltado para BAIXO, alimentado pelo Observador)
Y+1 B · O ← B = bloco de bambu Y+1, O = Observador (face→bambu, traseira→Pistão)
Y+0 B H · ← B = raiz do bambu, H = Funil (conectado ao Baú ao lado)
Y-1 · C · ← C = Baú (recebe itens do Funil)

Fio: Saída da face traseira do Observador → pó de Redstone → entrada do Pistão (se não estiverem diretamente adjacentes)

Sequência de Construção Passo a Passo

  1. Plante o bambu em um bloco de grama ou terra. Garanta ≥ 3 blocos de espaço vertical livre acima — o bambu cresce 1 bloco por vez até 16 blocos no máximo.
  2. Posicione o Pistão diretamente acima do bloco de bambu na altura Y+2, voltado para baixo. Quando acionado, ele se estenderá para baixo no bambu na altura 2, quebrando-o.
  3. Posicione o Observador adjacente ao caule do bambu na altura Y+1. A face (textura esculpida) deve apontar para o bloco de bambu. A traseira (lado liso) deve apontar para o Pistão — ou para o pó de Redstone que se conecta a ele.
  4. Conecte o Observador ao Pistão com pó de Redstone se eles não estiverem diretamente adjacentes. Lembre-se do máximo de 15 blocos — adicione um Repetidor se o fio for mais longo.
  5. Verifique a orientação: Clique com o botão direito no Observador e verifique se o indicador de saída dele se alinha com o fio. Uma verificação comum: coloque uma Lâmpada de Redstone temporária no fio e confirme se ela pulsa quando o bambu cresce.
  6. Adicione a coleta por Funil: Segure Shift e clique com o botão direito no bloco abaixo da raiz do bambu para colocar um Funil voltado para um Baú na lateral. O Funil coleta automaticamente os itens que caem e os deposita no Baú.
  7. Teste: Aguarde o bambu crescer um estágio. Sequência: o bambu cresce → o Observador detecta a mudança de estado → emite um pulso de 1 tick → o Pistão se estende → o bambu na altura 2 se quebra → solta os itens → o Funil coleta → os itens aparecem no Baú.
✅
Marco de sucesso: Itens se acumulando no Baú sem nenhuma ação do jogador. Essa é a primeira experiência dos alunos com uma automação autônoma real. Comemore isso explicitamente com um elogio específico e conectado ao comportamento.
Teoria Técnica — Fábrica de Pedregulho com Água + Lava

Química de Colisão — O que Acontece Exatamente e o Porquê

// Três interações possíveis entre Água × Lava:

Fonte de Água + Lava corrente → Pedregulho (Cobblestone) ← ALVO DO MÓDULO 1
Fonte de Lava + Água corrente → Obsidiana ← permanente; requer Picareta de Diamante
Fonte de Lava + Fonte de Água → nenhum bloco novo ← apenas limite de fluido; sem conversão

REGRA-CHAVE: Para o Pedregulho, a LAVA deve estar correndo, não ser um bloco de fonte.
Um espaço vazio entre os blocos de fonte força ambos a fluir antes de se encontrarem.

Layout de Construção

  1. Cave um canal de 3 blocos de largura e 1 bloco de profundidade em qualquer área plana.
  2. Coloque um bloco de fonte de Água na extremidade esquerda do canal (balde de água, clique com o botão direito no chão).
  3. Coloque um bloco de fonte de Lava na extremidade direita. Deve haver exatamente 1 bloco vazio de espaço entre os dois blocos de fonte — isso não é negociável.
  4. A Água flui para a direita; a Lava flui para a esquerda. Elas se encontram no espaço central, e a Lava corrente se solidifica em contato com o fluxo de Água, produzindo Pedregulho.
  5. Mine o bloco de Pedregulho com qualquer Picareta. Os dois fluxos se reencontram imediatamente no espaço agora vazio e geram um novo bloco de Pedregulho. Esse ciclo é infinito.
⚠️
Erro da Obsidiana: Os alunos colocam a fonte de Lava diretamente adjacente à fonte de Água (espaço zero). Isso produz Obsidiana, não Pedregulho, porque ambas são agora fontes estacionárias se tocando diretamente. A Obsidiana requer uma Picareta de Diamante para ser minerada e não pode ser quebrada no estágio do Módulo 1. Sempre reforce a regra do espaço de 1 bloco antes de os alunos colocarem a Lava.

Analogias Conceituais — Simplificando a Abstração para Idades de 8 a 9 Anos

💡 Redstone = "Eletricidade Residencial"
"O pó de Redstone é o fio dentro das paredes da sua casa. A alavanca ou o botão é o interruptor de luz. O Pistão é o motor ou a lâmpada. E assim como a eletricidade real só pode viajar uma certa distância antes que a voltagem caia demais, o sinal de Redstone decai após 15 blocos — é por isso que precisamos de um Repetidor, que funciona como um amplificador de potência em um cabo de caixa de som longo."

💡 Observador = "Alerta de Texto do Sensor de Movimento do Jardim"
"Imagine que você plantou um girassol e quisesse saber o segundo exato em que ele cresceu mais. Você poderia ficar olhando para ele o dia todo — ou colocar um sensor de movimento ao lado dele que lhe envia uma mensagem de texto no celular no instante em que ele se move. O Observador é esse sensor. No momento em que o bambu cresce um milímetro, ele envia uma mensagem de texto de Redstone para o Pistão."

💡 Coordenadas = "Sistema de Endereço Residencial da Cidade"
"Cada prédio em uma cidade tem um endereço: Rua Leste, 100, Quinta Avenida, Andar 3. No Minecraft, X é o número da sua rua no sentido Leste-Oeste, Z é o número da sua avenida no sentido Norte-Sul e Y é o andar em que você está. Sua fazenda de bambu tem um endereço — X=100, Y=64, Z=200 — e nenhum outro bloco em todo o mundo infinito compartilha esse endereço."

💡 Loop de Repetição = "A Fotocopiadora"
"Você escreveu a frase perfeita no papel e precisa de 20 cópias. Você poderia escrevê-la 20 vezes à mão — ou colocá-la em uma fotocopiadora e digitar '20'. O bloco Repetir é a sua fotocopiadora. Escreva a ação uma vez, diga de quantas cópias precisa e ela faz todas elas enquanto você relaxa."

💡 Agente = "Um Robô que Apenas Segue Instruções Escritas"
"O Agente é o seu melhor amigo robô — mas tem uma coisa: ele só pode fazer o que você escrever para ele com antecedência. Ele não sabe improvisar. Se você disser 'mova-se para a frente', ele se moverá para a frente. Se disser 'vire à direita, mova-se para a frente', ele virará à direita e depois se moverá para a frente. Ele seguirá suas instruções perfeitamente, mesmo que essas instruções o joguem de um penhasco. Por isso nós planejamos antes de programar." (Apresente como um personagem primeiro: "É um dinossauro? Um frango? Não — é o seu novo melhor amigo: o Agente!")

💡 Coordenadas Relativas vs. Absolutas — "GPS vs. Direções Pessoais"
"Coordenadas absolutas são como o GPS — a Rua Principal, 100 sempre será a Rua Principal, 100, não importa onde você esteja. Coordenadas relativas são como dar instruções a alguém a partir do local onde ela está agora — 'vire à esquerda e ande 3 blocos'. O til (~) significa 'a partir de onde estou parado agora'. É por isso que dois jogadores diferentes usando /tp @s ~3 ~0 ~0 terminam em dois lugares diferentes — ambos se moveram 3 blocos para o leste a partir de suas próprias posições."
