Módulo 4 de 5
🌲 Floresta Mecânica

Automação viva — os alunos aplicam todas as habilidades dos Módulos 1 a 3 para construir uma fazenda automatizada de ponta a ponta: o Agente planta as sementes, os circuitos de Redstone colhem quando as plantações estão prontas e os Funis organizam e armazenam a produção.

Objetivos do Módulo 4: Programar o Agente como um Jardineiro automatizado (loop + comandos de plantio) · Construir circuitos de colheita condicionais de Redstone (Observador → Pistão) · Implementar a organização e armazenamento de itens baseados em Funil · Compreender o fluxo de automação completo: Detectar → Ativar → Coletar → Armazenar.
1 · Visão Geral do Módulo — Integração & Escalada

Por que o Módulo 4 é o "Módulo de Integração"

Cada módulo anterior introduziu conceitos isolados. O Módulo 4 é o primeiro módulo que pede explicitamente aos alunos para combinar todas as habilidades anteriores em um único sistema:

  • Do Módulo 1: Controle do Agente + circuito de Redstone com Observador–Pistão
  • Do Módulo 2: Posicionamento por coordenadas para o layout da grade da fazenda
  • Do Módulo 3: Lógica condicional ("se a planta cresceu, colha-a")

O resultado é um sistema agrícola totalmente autônomo que roda sem a intervenção do jogador — a construção de maior complexidade no Nível 1. Os alunos que concluem o Módulo 4 com sucesso demonstraram a capacidade de projetar, construir e depurar um sistema automatizado multicomponente.

2 · O Agente como Jardineiro — Plantio Automatizado

Programando o Agente para Plantar em Fileiras

No Módulo 1, os alunos moveram o Agente manualmente com comandos básicos de movimento. O Módulo 4 usa o Agente para plantar sementes automaticamente em uma grade estruturada — o primeiro uso do Agente como uma ferramenta automatizada produtiva em vez de um brinquedo de demonstração.

// Carregue as sementes no inventário do Agente primeiro (clique com o botão direito no Agente, adicione Sementes ao slot 1)
on chat command "plant" {
  repeat 5 times { ← planta 5 sementes em uma fileira
    agent place on move down ← coloca o item abaixo enquanto o Agente se move
    agent move forward 1 ← avança um bloco
  }
}

Pré-condições críticas para que isso funcione:

  1. As sementes devem ser carregadas no inventário do Agente antes que o comando de chat dispare.
  2. O Agente deve estar posicionado no primeiro local de plantio, virado para a direção correta da fileira.
  3. O solo deve ser terra arada (use uma Enxada na terra/grama, depois molhe-a em um raio de até 4 blocks).
  4. O plantio do Agente usa "colocar ao mover para baixo", o que coloca o item no bloco diretamente abaixo do Agente à medida que ele avança.
⚠️
Falha mais comum no plantio do Agente: O Agente joga as sementes no chão em vez de plantá-las. Isso acontece quando o solo não é terra arada — sementes colocadas em blocos de terra comuns son tratadas como itens dropados, não como plantações semeadas. Sempre confirme se o solo está arado e a até 4 blocos de uma fonte de água antes de executar o comando de plantio.

Plantio em Várias Fileiras — Loops Aninhados

Para uma fazenda 5×5, os alunos precisam de loops aninhados: um loop externo para as fileiras e um loop interno para o comprimento de cada fileira. Essa é a introdução do Módulo 4 à iteração aninhada — um conceito que prefigura diretamente a varredura de matrizes 2D em etapas posteriores da educação em programação.

on chat command "farm" {
  repeat 5 times { ← loop externo: 5 fileiras
    repeat 5 times { ← loop interno: 5 colunas
      agent place on move down
      agent move forward 1
    }
    agent move right 1 ← desloca uma fileira para a direita
    agent turn around ← vira para a direção oposta para a próxima fileira
    agent move back 5 ← retorna ao início da fileira
  }
}
3 · Circuitos de Colheita Condicionais

O Circuito de Colheita com Observador–Pistão — Aplicado a Plantações

Os alunos já construíram uma cadeia de Observador–Pistão para bambu no Módulo 1. O Módulo 4 aplica o mesmo princípio ao trigo, cenouras, batatas e outras plantações cultiváveis. A principal diferença é a detecção do estágio de crescimento:

  • Bambu: cresce em incrementos distintos de blocos inteiros — fácil de detectar.
  • Trigo/Cenouras/Batatas: passam por 8 estágios de crescimento (0–7). O Estágio 7 é o crescimento completo. O Observador detecta a transição do estágio 6 → 7 como uma mudança de estado de bloco e dispara.
  • Repita o circuito do Módulo 1: face do Observador → fileira de cultivo. Parte traseira do Observador → entrada do Pistão (ou fio de Redstone). O Pistão se estende horizontalmente para quebrar a fileira de plantação madura.
💡
Ensinando a condicional "crescer para colher": Enquadre como uma estrutura se-então, mesmo que seja implementada em hardware: “SE o trigo atingir o estágio de crescimento 7 (totalmente crescido), ENTÃO o Observador dispara, ENTÃO o Pistão quebra as plantações, ENTÃO as sementes e os itens de comida caem.” Essa linguagem une a lógica condicional do MakeCode do Módulo 3 à lógica condicional do Redstone do Módulo 4.
4 · Mecânicas do Funil & Sistemas de Ordenação de Itens

O que é um Funil (Hopper)?

Um Funil é um bloco em forma de funil com duas funções: (1) ele coleta itens que caem nele por cima e (2) transfere itens de seu próprio inventário para um contêiner adjacente (baú, fornalha, outro Funil) ou os deponha para baixo através de sua boca de saída.

Regras de Posicionamento do Funil

  • Direção de saída: determinada por qual bloco o Funil está voltado — sua rampa de saída aponta para o bloco no qual ele foi colocado.
  • Para conectar a um baú: segure Shift + clique com o botão direito no baú. Sem o Shift, o inventário do baú se abrirá em vez de posicionar o Funil.
  • Para encadear Funis: coloque-os de ponta a ponta; os itens são transferidos pelo encadeamento a uma taxa fixa de 2,5 itens por segundo por Funil.
  • Capacidade: cada Funil comporta 5 pacotes (stacks) de itens (até ×64 por pacote).

Sistema Básico de Ordenação

Uma ordenação simples de dois baús: um Funil acima de cada baú, cada um carregado com um item de filtro (o item que você deseja rotear para aquele baú específico). Quando os itens fluem, os itens que correspondem ao filtro vão para o baú daquele Funil; itens que não correspondem passam para o próximo Funil.

Objetivo do Módulo 4: Direcionar o trigo para um baú e as sementes para outro — separando a produção de alimentos do material de replantio de forma automática.

Tempo de Transferência do Funil — Números Práticos

Taxa de transferência: um Funil processa um item a cada 4 ticks de jogo (0,2 segundos). A 20 ticks/segundo, isso equivale a 2,5 itens por segundo. Para uma fazenda grande que produz altos volumes, um único encadeamento de Funis pode se tornar um gargalo. Soluções:

  • Use encadeamentos de Funis paralelos — múltiplos pontos de coleta, cada um alimentando seu próprio baú.
  • Use um Carrinho com funil (Hopper Minecart) sobre um trilho de coleta — carrinhos com funil transferem itens com o dobro da velocidade de Funis estacionários.
5 · O Fluxo Completo de Automação

O Fluxo de Quatro Etapas — Detectar → Ativar → Coletar → Armazenar

A estrutura de ensino assinada do Módulo 4. Todo sistema de fazenda totalmente automatizado pode ser descrito em quatro etapas sequenciais:

Etapa 1: Detectar

O bloco do Observador monitora o bloco da plantação para identificar uma mudança de estado (estágio de crescimento 6 → 7). Quando detectada, ele emite um pulso instantâneo de Redstone de 1 tick.

Etapa 2: Ativar

O pulso do Observador alimenta o Pistão via fio de Redstone. O Pistão se estende, quebrando o bloco de plantação totalmente crescido. A plantação cai como entidades de itens no chão.

Etapa 3: Coletar

O Funil posicionado abaixo da fileira de cultivo coleta as entidades de itens que caem. Os itens entram no inventário de 5 slots do Funil automaticamente dentro de 0,5 segundos após entrarem no alcance do Funil.

Etapa 4: Armazenar

O Funil transfere os itens coletados para o Baú conectado a uma taxa de 2,5 itens por segundo. O sistema de ordenação direciona diferentes tipos de itens para baús diferentes com base nos filtros do Funil.

✅
Momento de ensino: Assim que todas as quatro etapas forem construídas e testadas, peça para os alunos descreverem o fluxo com suas próprias palavras sem olhar para o código: "O que detecta? O que ativa? O que coleta? O que armazena?" Essa codificação verbal solidifica a estrutura conceitual e os prepara para o trabalho de design de projetos do Módulo 5.
6 · Catálogo de Resolução de Problemas — Módulo 4 (5 Cenários)
Erro / SintomaCausa RaizSolução ExataRoteiro de Prevenção
O Agente joga as sementes no chão em vez de plantá-las O solo abaixo do Agente é terra ou grama não arada, não terra arada. Sementes colocadas em blocos que não sejam de terra arada aparecem como entidades de itens soltos (itens dropados), não como plantações semeadas. Antes de rodar o comando de plantar: (1) Use uma Enxada em cada bloco de terra para transformá-lo em terra arada. (2) Coloque uma fonte de água a até 4 blocos da terra arada para hidratá-la (castanho-escuro = hidratado, bege-claro = seco — as plantações crescem muito mais devagar em terra arada seca). (3) Confirme executando o Agente primeiro sobre uma terra arada já preparada. "Antes que o Agente possa plantar, o solo deve estar arado e regado. Verifique a cor do solo — castanho-escuro = pronto para plantar."
O Funil se enche de itens, mas não os transfere para o baú A rampa de saída do Funil não está apontando para o baú — ele foi colocado sem segurar o Shift, fazendo com que o inventário do baú se abrisse em vez de criar uma conexão. O Funil agora está apontando para baixo no ar. Quebre o Funil e recoloque-o: segure o Shift e clique com o botão direito diretamente no bloco do baú. Confirme visualmente se a saída do Funil se conecta à lateral do baú. Alternativamente, quebre o Funil e recoloque-o adjacente ao baú em uma orientação horizontal (o baú pode ficar na lateral, não apenas abaixo). "Sempre segure o Shift ao colocar um Funil em um baú. Sem Shift = o baú abre = o Funil fica posicionado de forma errada."
O Observador dispara quando as plantações crescem, mas o Pistão não se estende — as plantações nunca são colhidas A face de saída do Observador não está conectada ao Pistão. Esse é o mesmo problema de orientação do Módulo 1, mas costuma ser mais confuso porque o layout da fazenda é mais complexo. Teste em isolamento: desconecte o Observador do circuito da fazenda. Coloque uma Lâmpada de Redstone diretamente atrás da face de saída do Observador. Se a Lâmpada acender com o crescimento da plantação, o Observador funciona corretamente e o problema está adiante (lacuna no fio ou orientação do Pistão). Corrija a lacuna ou recoloque o Pistão voltado para a direção correta. "Teste cada etapa separadamente. O Observador funciona? Então verifique o fio. O fio funciona? Then check the Piston orientation."
O loop aninhado de plantio em fileiras do Agente planta as culturas em espiral ou padrão aleatório em vez de uma grade O direcionamento do Agente após o "dar a volta" não está orientado corretamente — o aluno usou "virar à esquerda" duas vezes, mas esqueceu que após um número ímpar de comandos "virar à direita", "virar à esquerda + virar à esquerda" ≠ curva de 180° em todos os casos. Ou a contagem de passos no movimento de retorno está errada. Desenhe o caminho do Agente no papel antes de programar. Marque cada curva explicitamente. Verifique: após concluir a fileira 1, o Agente está voltado para a direção oposta? O comando "mover para trás 5" o retorna exatamente ao início da fileira? Teste primeiro com 1 fileira, observe e depois estenda para 5 fileiras. "Desenhe o caminho antes de programá-lo. Qual direção após a curva 1? Após a curva 2? O Agente não sabe para onde 'deveria' ir — apenas para onde seu código manda ir."
O sistema de ordenação direciona TODOS os itens para o primeiro baú, independentemente do tipo de item Os slots do inventário do Funil de filtragem não estão configurados corretamente — os itens de filtro devem preencher parcialmente um slot no inventário do Funil para que apenas itens correspondentes sejam retidos ali. Alternativamente, nenhum filtro foi definido e o primeiro Funil pega todos os itens indiscriminadamente. A ordenação por Funil exige a colocação de um item de filtro em cada slot do inventário do Funil filtrador, sem deixar slots vazios. Os itens que correspondem ao filtro preenchem os slots de filtragem; os itens que não correspondem passam direto. Uma simplificação comum: use Funis Bloqueados com itens específicos — isso requer um circuito comparador de Redstone. Para o Módulo 4, use o método de filtro visual mais simples e teste com dois itens do tipo (trigo vs. sementes) para confirmar o roteamento. "O Funil de ordenação precisa de um 'modelo' — coloque um de cada item desejado em seus slots de inventário. Ele retém os itens correspondentes e deixa os outros passarem."
7 · Dicas Pro do Tutor & Analogias do Mundo Real

💡 Organização por Funil = A Analogia da "Usina de Reciclagem"
“Pensem em uma usina de reciclagem: diferentes esteiras transportadoras direcionam garrafas de vidro para uma caçamba, papel para outra e metal para outra. O sistema de funis na fazenda de vocês é a sua usina de reciclagem digital. O trigo vai para a esquerda, as sementes para a direita — de forma automática, todas as vezes, sem que ninguém precise organizar manualmente.”

💡 O Fluxo de Quatro Etapas = A Analogia do "Armazém da Amazon"
“Quando vocês compram algo na Amazon: (1) Um sensor digital detecta o pedido de vocês [Observador]. (2) Um braço robótico retira o item da prateleira [Pistão]. (3) Uma esteira transportadora o leva até a estação de embalagem [Funil]. (4) O pacote vai para uma caixa identificada na seção correta do depósito [Baú]. Nossa fazenda funciona nessas mesmas quatro etapas, apenas processando trigo em vez de pacotes.”

💡 Loops Aninhados = A Analogia da "Impressora" para Grades 2D
“Quando uma impressora imprime uma página de texto, a cabeça se move horizontalmente imprimindo uma linha de pontos (loop interno), depois desce uma linha (loop externo) e imprime a linha seguinte, repetindo esse processo até preencher o papel. O plantio em fileiras feito pelo Agente funciona de forma idêntica: o loop interno é uma fileira de sementes, e o loop externo realiza a mudança de fileira.”

💡 Testando em Isolamento — A Regra de "Depurar uma Etapa por Vez"
Para sistemas multicomponentes, sempre teste cada etapa de forma isolada antes de integrá-las ao circuito final. O protocolo de depuração para o Módulo 4 assume: (1) Verificar se o Observador envia o pulso com o crescimento da plantação (use uma lâmpada). (2) Verificar se o Pistão se estende quando alimentado manualmente (use uma Alavanca). (3) Verificar se o Funil coleta os itens (jogue itens manualmente sobre ele). (4) Verificar se o Funil direciona os itens para o baú correto (coloque itens de teste no filtro). Somente quando cada etapa passar no teste com sucesso os alunos devem conectar o fluxo completo.
