🧩 Oasis Logika

Permainan, teka-teki, dan fondasi nyata logika pemrograman — siswa membangun mini-game interaktif pertama mereka menggunakan blok peristiwa MakeCode, variabel, dan kondisi.

Tujuan Modul 3: Pahami pemrograman berbasis peristiwa (Saat Pemain Berjalan, Saat Blok Dihancurkan) · Buat dan ubah variabel sebagai "wadah skor" · Terapkan logika kondisional Jika–Maka dan Jika–Maka–Selain · Gabungkan Peristiwa + Variabel + Kondisi untuk membangun mini-game yang dapat dimainkan.
1 · Gambaran Umum — Titik Balik Pemrograman

Mengapa Modul 3 adalah Titik Balik Konseptual

Modul 1 dan 2 menggunakan MakeCode untuk mengotomatisasi konstruksi fisik — program berjalan sekali dan menghasilkan hasil statis. Modul 3 memperkenalkan pemrograman reaktif dan interaktif: kode yang menunggu sesuatu terjadi, lalu merespons. Ini adalah paradigma mendasar dari hampir semua perangkat lunak dunia nyata. Siswa sekarang tidak membangun mesin — mereka membangun pengalaman.

2 · Pemrograman Berbasis Peristiwa

Apa itu Peristiwa?

Sebuah peristiwa adalah sinyal bahwa sesuatu telah terjadi di dunia game. Sebuah blok peristiwa di MakeCode mendaftarkan pendengar: kode yang tertutup di dalamnya hanya berjalan tepat saat peristiwa itu terpicu.

saat pemain berjalan { ← blok peristiwa: terpicu setiap langkah
  ubah skor sebesar 1 ← kode ini berjalan setiap peristiwa terpicu
} ← kode DI LUAR blok ini = tidak dipicu oleh jalan

Peristiwa Modul 3 yang perlu diajarkan tutor:

  • Saat Pemain Berjalan: Terpicu sekali per langkah yang diambil pemain.
  • Saat Blok Dihancurkan: Terpicu saat pemain menambang jenis blok tertentu.
  • Pada Perintah Obrolan [kata]: Terpicu saat pemain mengetik kata di obrolan.
  • Saat Pemain Melompat: Terpicu sekali per lompatan.
⚠️
Kesalahan Kritis: Kode yang Ditempatkan di Luar Blok Peristiwa
Ini adalah kesalahan coding paling umum di Modul 3. Siswa menempatkan blok "ubah variabel sebesar 1" di luar blok peristiwa. Hasilnya: variabel hanya berubah saat program dimuat, bukan saat peristiwa terpicu.
SALAH — skor berubah saat program dimuat, bukan saat berjalan:
saat pemain berjalan { ← blok peristiwa kosong
}
ubah skor sebesar 1 ← DI LUAR — berjalan sekali, tidak pernah lagi
BENAR — skor berubah setiap langkah:
saat pemain berjalan {
  ubah skor sebesar 1 ← DI DALAM
}
Koreksi pengajaran: Sebelum coding, minta siswa menggambar persegi panjang di kertas dan secara fisik memutuskan instruksi mana yang masuk ke dalam dan luar.
3 · Variabel — Wadah untuk Data yang Berubah

Apa itu Variabel?

Sebuah variabel adalah lokasi penyimpanan memori bernama yang menyimpan sebuah nilai. Semua variabel di Modul 3 bersifat numerik — menyimpan skor, penghitung, dan timer.

atur skor ke 0 ← membuat variabel dan memberi nilai awal
ubah skor sebesar +1 ← menambahkan 1 ke nilai yang disimpan saat ini
ubah skor sebesar -5 ← mengurangi 5 dari nilai saat ini
katakan skor ← menampilkan nilai saat ini di obrolan (untuk pengujian)

Cakupan variabel di MakeCode: Semua variabel bersifat global — blok peristiwa mana pun dapat membaca dan menulis variabel mana pun. Tidak ada cakupan variabel lokal.

Konvensi penamaan yang harus diajarkan: Gunakan nama deskriptif (skor, langkah, batu) daripada huruf tunggal. Ini memungkinkan siswa usia 8 tahun untuk men-debug kode mereka sendiri.

4 · Kondisi — Logika Jika–Maka

Blok Jika–Maka dan Jika–Maka–Selain

jika skor ≥ 10 maka {
  katakan "Menang! Skor akhir: " + skor
  kembang api di posisi pemain
  atur skor ke 0 ← reset untuk permainan baru
}

jika skor ≥ 10 maka {
  katakan "Menang!"
} selain {
  katakan "Terus! " + (10 - skor) + " langkah lagi"
}

Operator Perbandingan

= sama dengan · ≠ tidak sama dengan
> lebih besar · < lebih kecil
≥ lebih besar atau sama · ≤ lebih kecil atau sama

Operator Logika

DAN — kedua kondisi harus benar
ATAU — kondisi mana saja benar
TIDAK — membalikkan kondisi
Contoh: skor ≥ 10 DAN nyawa > 0

5 · Pembangunan Mini-Game — Contoh Lengkap

Mini-Game "Berlari ke 10" — Spesifikasi Teknis Lengkap

// Blok setup — berjalan sekali saat program dimuat
saat mulai {
  atur skor ke 0
  katakan "Game dimulai! Jalan untuk mendapat poin. Capai 10!"
}

// Blok peristiwa — terpicu setiap langkah
saat pemain berjalan {
  ubah skor sebesar 1
  jika skor ≥ 10 maka {
    katakan "MENANG! Skor akhir: " + skor
    kembang api di posisi pemain
    atur skor ke 0
  }
}

7 · Tips Pro & Analogi Dunia Nyata

💡 Variabel = Analogi "Skor Video Game"
"Pikirkan setiap video game yang pernah kamu mainkan. Mario punya penghitung koin. FIFA punya penghitung gol. Semua itu adalah variabel — kotak bernama yang menyimpan angka dan berubah saat kamu bermain. Hari ini kamu akan membangun kode di balik penghitung skormu sendiri."

💡 Peristiwa = Analogi "Sistem Notifikasi"
"Ponselmu tidak membaca setiap pesan yang dikirim saat itu juga. Ia diam menunggu — lalu memberi notifikasi saat pesan tiba. Notifikasi itu adalah peristiwa. Blok peristiwa MakeCode bekerja persis sama: tidur sampai sesuatu terjadi di Minecraft, lalu bangun dan jalankan kodenya."

💡 Jika–Maka = Analogi "Termostat"
"Termostat tidak terus memanaskan rumah — ia terus memeriksa: 'Apakah suhu di bawah 20°C?' Jika ya, pemanas menyala. Jika tidak, tetap mati. Blok Jika–Maka melakukan persis itu: memeriksa kondisi dan hanya bertindak jika kondisi benar."

💡 Teknik "Visualisasi Batas Blok" untuk Mencegah Kesalahan Blok Luar
Sebelum siswa mana pun menulis kode di Modul 3, minta mereka menggambar persegi panjang di kertas. Tulis "Saat Pemain Berjalan" di atasnya. Kemudian: "Instruksi mana yang masuk DI DALAM persegi panjang ini, mana yang di luar?" Siswa yang menggambar batas sebelum coding melakukan kesalahan ini ~70% lebih jarang.
