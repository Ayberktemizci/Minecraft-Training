Module 1: Brave New Green Stone World — the complete technical reference for everything you need to teach, explain, demonstrate, and troubleshoot. Pure mechanics only; no pedagogy lives here.

Module 1 Technical Objectives: Master X/Y/Z 3D coordinate system and /tp command syntax (relative ~ and absolute) · Open and navigate the MakeCode overlay via /code · Spawn, orient, and command the Agent robot · Understand Redstone signal strength and 15-block decay physics · Build the Observer→Piston automated bamboo farm with Hopper collection · Build the Water+Lava infinite cobblestone factory.
Module 1 Overview — Scope & Prerequisite Context

What Students Must Leave With After Module 1

Module 1 is the entry point of the entire Level 1 spiral. Every concept introduced here is directly prerequisite for subsequent modules — coordinates for Module 2, MakeCode loops for Modules 3 and 4, Redstone logic for Module 4. Students leave Module 1 able to: navigate in 3D without getting lost, spawn and command the Agent, understand Redstone as a signal system, and demonstrate one working automated build. These are not aspirational outcomes — they are structural requirements for Module 2 to function.

Pre-Session Technical Checks Specific to Module 1

Before the session launches, verify these Module 1-specific items (in addition to the standard Pre-Flight from the Pedagogy page):

  1. Open the session world and type /code in chat. If the MakeCode overlay does not appear, go to world settings and enable "Code Builder" before students join.
  2. Confirm the bamboo farm build zone has at least 3×3 clear ground tiles and 5 blocks of vertical clearance. Pre-build one Observer and one Piston as a demo reference.
  3. Prepare a 3-block-wide trench for the cobblestone factory demo. Pre-place the Water source at one end so students only need to place the Lava.
  4. Pin today's spawn coordinates in your own chat window and in the Zoom/Meet chat before students arrive.

Technical Theory — 3D Coordinate System (X, Y, Z)

Axis Definitions & World Reference Frame

Minecraft uses a right-handed 3D Cartesian coordinate system. Every block in the infinite world occupies a unique position defined by three signed integers:

X-axis → East (+X) / West (−X) ← horizontal east-west position
Y-axis → Up (+Y) / Down (−Y) ← ALTITUDE — height above bedrock
Z-axis → South(+Z) / North (−Z) ← horizontal north-south position

Bedrock layer = Y 0 (indestructible in Survival)
Sea level = Y 63 (default Overworld superflat)
Sky build limit = Y 255 (older editions) / Y 320 (1.18+)

How to read current position:
· Java Edition: press F3 — XYZ displayed top-left
· Minecraft Education Edition: Settings → World → enable "Show Coordinates" in HUD

Absolute Coordinates — Fixed Addresses in the World

// Absolute — teleports to a fixed world address regardless of current position
/tp @s 100 64 200 → self to X=100, Y=64, Z=200 (always)
/tp @a 0 64 0 → all to world origin exactly
/tp @a [X] [Y] [Z] → all to today's spawn coords (pin in chat)

Relative Coordinates — The ~ Tilde Prefix

The tilde means "offset from my current position right now." It is evaluated dynamically at the moment the command runs — not a fixed address. Relative offsets can be positive or negative:

// ~ = "from where I am now"
/tp @s ~0 ~0 ~0 → zero offset = no movement (stays put)
/tp @s ~3 ~0 ~0 → 3 blocks East from current X position
/tp @s ~0 ~5 ~0 → 5 blocks straight UP
/tp @s ~0 ~-3 ~0 → 3 blocks DOWN

// Mixed syntax — valid and frequently useful:
/tp @s ~0 64 ~0 → keep current X and Z; snap altitude to Y=64 exactly
/tp @s 100 ~0 ~0 → snap to X=100; keep current Y and Z

COMMON MISCONCEPTION: "/tp @a ~0 ~0 ~0 moves all players to the same spot."
CORRECT: "~0 ~0 ~0 = zero offset from EACH player's OWN position = no one moves."
💡
Teaching the difference physically: "Absolute coordinates are like a street address — 100 Main Street is always 100 Main Street. Relative coordinates are like directions — '3 blocks to your left' depends on which way you're currently facing."
Technical Theory — MakeCode Overlay Interface

Accessing MakeCode from Within Minecraft

Access path: Press T to open Minecraft chat → type /code → press Enter. The MakeCode editor opens as a full-screen overlay. Code is persistent between opens — closing and re-opening shows the same programme state.

Critical pre-condition: "Enable Code Builder" must be toggled ON in world settings (the pencil icon on the world thumbnail in the Worlds list) before any student enters the world. If this toggle is OFF, /code runs silently with no feedback — students see nothing and no error message. This is the single most common pre-session failure.

MakeCode Interface Zones — What Students Need to Know in Module 1

Left Panel — Block Palette

Categorised block library. Module 1 categories used: Loops (Repeat), Agent (move, turn, place), Player (On Chat Command). Blocks are dragged onto the canvas. Right-clicking a placed block reveals rename/delete options.

Right Panel — Coding Canvas

Where blocks are assembled. The canvas is infinite — zoom with scroll wheel. Blocks must be connected (snapped together) to run; floating blocks execute nothing. Green "play" button runs the programme. Red "stop" halts all running loops.

Core Block Types — Module 1 Required Knowledge

// 1. ON CHAT COMMAND — Event handler. Runs enclosed code when player types the word in MC chat.
on chat command "build" {
  // Code here runs ONLY when player types "build" in Minecraft chat
  // Code OUTSIDE this block runs once at program load only
}

// 2. REPEAT LOOP — Executes enclosed code exactly N times sequentially.
repeat 5 times {
  agent move forward 1 // runs 5 times total
}

// 3. AGENT MOVE — Moves Agent N steps in a direction relative to Agent's CURRENT FACING.
agent move forward 3 // 3 blocks in Agent's forward direction
agent move back 1 // 1 block behind Agent
agent move left 2 // 2 blocks to Agent's left (relative to facing)
agent turn left // rotates Agent 90° counter-clockwise — changes ALL subsequent directions

Agent Mechanics — Orientation, Inventory, and the Most Critical Concept

Every Agent movement command is evaluated relative to the Agent's current facing direction — not relative to the world compass. This facing changes after every Turn command. Students who do not explicitly track this will produce diagonal or incorrect paths every time.

// Agent starts facing NORTH (−Z direction)
agent move forward 3 → moves NORTH 3 blocks (Z decreases by 3)
agent turn right → Agent now faces EAST (+X)
agent move forward 2 → moves EAST 2 blocks (NOT north anymore!)
agent turn right → Agent now faces SOUTH (+Z)
agent move forward 1 → moves SOUTH 1 block

RULE: After every Turn block, "forward" means a different world direction.
Track facing explicitly. Draw on paper before coding any multi-turn sequence.
⚠️
Pre-coding requirement: Before any student writes Agent movement code with turns, ask them to draw a top-down map showing: (a) Agent start position, (b) initial facing arrow, (c) each turn with an updated arrow. Students who draw first code correctly ~75% of the time; those who code directly produce errors ~65% of the time.

Agent Inventory

The Agent has a 27-slot inventory (3×9 grid), identical in capacity to a chest. Items must be loaded into the Agent before it can place them. Loading: stand next to the Agent and right-click it to open its inventory GUI, then drag items in from your own inventory or a nearby chest.

// Agent place block — requires items pre-loaded in Agent inventory
agent place on move down // places item below Agent as it moves (for farming rows)
agent place forward // places item 1 block in front of Agent
agent place down // places item directly below current position

// Common error: Agent does nothing when place is called.
// Cause: inventory slot is empty. Always load before running place commands.
Technical Theory — Redstone Signal Physics

Signal Strength, Propagation, and 15-Block Decay

Every Redstone power source emits a signal at strength 15. Each block of Redstone dust the signal travels through decrements strength by exactly 1. At strength 0, the signal dies — the circuit is broken.

// Signal strength vs. distance (no Repeater)
Dust[1]: str=15 → Dust[5]: str=11 → Dust[10]: str=6
→ Dust[14]: str=2 → Dust[15]: str=1 → Dust[16]: str=0 → DEAD

Maximum wire length before Repeater: 15 blocks

Fix: Place a Redstone Repeater at or before block 15.
The Repeater resets signal strength to 15 regardless of incoming strength.
Repeaters also add configurable delay: 1–4 ticks (1 tick = 0.05 seconds).

The Four Components Students Must Know in Module 1

Observer Block

Detects block state changes in the single block its face (carved texture side) directly looks at. Emits a 1-tick Redstone pulse when a state change is detected. For bamboo: growing from one stage to the next is a block state change. The Observer does not emit a sustained signal — only a brief pulse.

Two sides to know: The face (carved texture) must point at the bamboo. The back (output, smooth side) must point toward the Piston or connecting wire. Getting either wrong breaks the circuit silently.

Piston (Regular)

Extends 1 block when powered; retracts cleanly when de-powered. Used to break bamboo: the Piston head extends into the bamboo block, breaking it and causing items to drop as entities.

Regular vs. Sticky: A Sticky Piston pulls the adjacent block back when de-powered. For bamboo harvesting, use a regular Piston — it breaks the bamboo on extend and retracts clean, ready for the next growth cycle.

Button vs. Lever

Button (Stone): Momentary pulse of ~0.5 seconds, then automatically returns to off. Best for one-shot triggers. Lever: Continuous signal until manually flipped back. Best for states that must remain on/off (e.g., a gate that should stay open).

Knowing this distinction prevents the common error of using a Button where a sustained signal is required — e.g., trying to keep a Redstone Lamp lit with a Button (it pulses and goes dark).

Redstone Repeater

Resets signal back to strength 15. Adds 1–4 tick delay (right-click to cycle). Directional: must be placed in-line with the signal path, not perpendicular. Common error: Repeater placed across the wire at 90° — it receives no signal and transmits nothing, but this failure is visually non-obvious.

Technical Theory — Automatic Bamboo Farm (Build Blueprint)

Component Map Before Building

// Vertical cross-section view — X = block, · = air

Y+3 · · ·
Y+2 · · · ← Piston (facing DOWN, powered by Observer)
Y+1 B · O ← B = bamboo block Y+1, O = Observer (face→bamboo, back→Piston)
Y+0 B H · ← B = bamboo root, H = Hopper (connected to Chest beside it)
Y-1 · C · ← C = Chest (receives items from Hopper)

Wire: Observer back-face output → Redstone dust → Piston input (if not directly adjacent)

Step-by-Step Build Sequence

  1. Plant bamboo on a grass or dirt block. Ensure ≥ 3 blocks of vertical clearance above — bamboo grows 1 block at a time up to 16 blocks maximum.
  2. Place the Piston directly above bamboo block at height Y+2, facing downward. When triggered, it will extend down into bamboo at height 2, breaking it.
  3. Place the Observer adjacent to the bamboo stalk at height Y+1. The face (carved texture) must point at the bamboo block. The back (smooth side) must point toward the Piston — or toward Redstone dust connecting to it.
  4. Connect Observer to Piston with Redstone dust if they are not directly adjacent. Remember the 15-block maximum — add a Repeater if the wire is longer.
  5. Verify orientation: Right-click the Observer and check that its output indicator aligns with the wire. A common check: place a temporary Redstone Lamp on the wire and confirm it pulses when bamboo grows.
  6. Add Hopper collection: Hold Shift and right-click the block below the bamboo root to place a Hopper facing into a Chest to the side. The Hopper automatically collects fallen items and deposits them into the Chest.
  7. Test: Wait for bamboo to grow one stage. Sequence: bamboo grows → Observer detects state change → emits 1-tick pulse → Piston extends → bamboo at height 2 breaks → drops items → Hopper collects → items appear in Chest.
✅
Success milestone: Items accumulating in the Chest with no player action. This is students' first experience of genuine autonomous automation. Celebrate it explicitly with specific, behaviour-linked praise.
Technical Theory — Water + Lava Cobblestone Factory

Collision Chemistry — Exactly What Happens and Why

// Three possible Water × Lava interactions:

Water source + flowing Lava → Cobblestone ← MODULE 1 TARGET
Lava source + flowing Water → Obsidian ← permanent; requires Diamond Pickaxe
Lava source + Water source → no new block ← fluid boundary only; no conversion

KEY RULE: For Cobblestone, the LAVA must be flowing, not a source block.
One empty gap between source blocks forces both to flow before meeting.

Build Layout

  1. Dig a channel 3 blocks wide, 1 block deep in any flat area.
  2. Place a Water source block at the far-left end of the channel (bucket of water, right-click the floor).
  3. Place a Lava source block at the far-right end. There must be exactly 1 empty block gap between the two source blocks — this is non-negotiable.
  4. The Water flows right; the Lava flows left. They meet in the centre gap, and the flowing Lava solidifies on contact with the Water flow, producing Cobblestone.
  5. Mine the Cobblestone block with any Pickaxe. The two flows immediately re-meet in the now-empty gap and generate a new Cobblestone block. This cycle is infinite.
⚠️
Obsidian mistake: Students place the Lava source directly adjacent to the Water source (zero gap). This produces Obsidian, not Cobblestone, because both are now stationary sources touching directly. Obsidian requires a Diamond Pickaxe to mine and cannot be broken at Module 1 stage. Always enforce the 1-block gap rule before students place Lava.

Concept Analogies — Simplify Abstraction for Ages 8–9

💡 Redstone = "House Electricity"
"Redstone dust is the wire inside your house walls. The lever or button is the light switch. The Piston is the motor or light bulb. And just like real electricity can only travel a certain distance before the voltage drops too low, Redstone signal decays after 15 blocks — that's why we need a Repeater, which works like an amplifier booster in a long speaker cable."

💡 Observer = "Garden Motion Sensor Text Alert"
"Imagine you planted a sunflower and wanted to know the exact second it grew taller. You could stare at it all day — or put a motion sensor next to it that sends you a text message the instant it moves. The Observer is that sensor. The moment bamboo grows one millimetre, it fires a Redstone text message to the Piston."

💡 Coordinates = "City Street Address System"
"Every building in a city has an address: 100 East, 5th Street, Floor 3. In Minecraft, X is your East-West street number, Z is your North-South avenue number, and Y is the floor you're on. Your bamboo farm has an address — X=100, Y=64, Z=200 — and no other block in the entire infinite world shares that address."

💡 Repeat Loop = "The Photocopier"
"You wrote the perfect sentence on paper and need 20 copies. You could write it 20 times by hand — or put it in a photocopier and press '20'. The Repeat block is your photocopier. Write the action once, tell it how many copies you need, and it does all of them while you sit back."

💡 Agent = "A Robot That Only Follows Written Instructions"
"The Agent is your robotic best friend — but here's the thing: it can only do what you write down for it in advance. It can't improvise. If you say 'move forward', it moves forward. If you say 'turn right, move forward', it turns right then moves forward. It will follow your instructions perfectly, even if those instructions send it off a cliff. That's why we plan before we code." (Introduce as a character first: "Is it a dinosaur? A chicken? No — it's your new best friend: the Agent!")

💡 Relative vs. Absolute Coordinates — "GPS vs. Personal Directions"
"Absolute coordinates are like GPS — 100 Main Street is always 100 Main Street, no matter where you are. Relative coordinates are like giving someone directions from where they're standing right now — 'turn left and walk 3 blocks'. The tilde (~) means 'from where I'm standing right now'. That's why two different players using /tp @s ~3 ~0 ~0 end up in two different places — both moved 3 blocks east from their own positions."
